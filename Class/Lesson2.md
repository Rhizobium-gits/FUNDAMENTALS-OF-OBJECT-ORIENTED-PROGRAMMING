# 授業まとめ：抽象化とオブジェクト指向プログラミングの基礎

## 1. 抽象化の概念とメリット
授業の前半では、プログラミングにおける「抽象化」の重要性が、関数の復習を交えて説明されました。

*   **抽象化とは**：複雑な仕組み（内部処理）を隠し、結果や目的に注力できるようにすることです。
    *   **車の運転の例え**：運転手はアクセル、ブレーキ、ハンドルの操作といった細かい仕組みを意識せず、「ドライブに行く」「買い物に行く」という上位レベルの目的に意識を向けることができます。これが「身体化」された抽象化の状態です。
    *   **ギタリストの例え**：機材がどう動いているか知らなくても「ロックする」という目的に集中できる状態は、非常にオブジェクト指向的な思考と言えます。
*   **関数の役割**：関数に名前（例：`drawAnpanman`）を付けることで、その中身の複雑さ（線の太さ、色、座標計算など）を忘れることができます。これにより、プログラムの再利用性が高まり、次のステップへの試行錯誤（トライアンドエラー）が容易になります。

## 2. オブジェクト指向プログラミング (OOP) の導入
1990年頃から普及し始めた考え方で、関連する「データ（変数）」と「振る舞い（関数）」を一まとめにするコーディング手法です。

### 基本用語と概念
授業では、以下のカタカナ用語を概念として理解することが強調されました。

| 用語 | 説明 | 例え（設計図と実体） |
| :--- | :--- | :--- |
| **クラス (Class)** | オブジェクトの設計図、または集合の定義。 | ケーキのレシピ、車の設計図、犬という概念。 |
| **インスタンス (Instance)** | クラス（設計図）から実体化したもの。**オブジェクト**とも呼ばれます。 | 実際に焼かれたケーキ、特定の車（ホンダ・ヴェゼルなど）、ポチ（特定の犬）。 |
| **データフィールド (Field)** | オブジェクトが持つ状態や属性（変数）。 | リンゴの色・重さ、車の現在の座標(X, Y)。 |
| **メソッド (Method)** | オブジェクトが行う振る舞い（関数）。 | リンゴが「熟す」、車が「加速する」、アンパンマンを「描画する」。 |
| **コンストラクター** | インスタンスを生成する際に呼び出される特別なメソッド。 | ジャムおじさんがアンパンマンの顔を焼くプロセス。 |

## 3. Processingによる実装の実際
アンパンマンをマウスに追従させるプログラムを例に、具体的な書き換え手順が示されました。

*   **クラスの定義**：`class Anpanman { ... }` の中に変数と関数をまとめます。
*   **インスタンスの生成**：`new` キーワードを使い、設計図から実体を作ります。
    *   例：`Anpanman boku = new Anpanman();`
*   **メッセージの送信（命令）**：ドット演算子を使って、インスタンスに命令を出します。
    *   例：`boku.display();`
*   **カプセル化（隠蔽）**：外部から直接変数を操作するのではなく、メソッド（例：`display()`）を呼び出すだけで済むように作ると、よりオブジェクト指向らしくなります。

## 4. 重要な注意点とエラー
### スコープ (Scope)
変数が認識される範囲のことです。
*   インスタンスを保持する変数は、`setup()` や `draw()` の外側（グローバルスコープ）で定義しないと、関数をまたいでアクセスできません。

### NullPointerException (ぬるぽ)
実体（インスタンス）がないのに命令を送ろうとした時に発生するエラーです。
*   **トイレットペーパーの例え**：ホルダー（変数）はあるが、中身（インスタンス）が空の状態が「null」です。その状態で紙を使おうとするとエラー（絶望）になります。
*   対策として、`if (boku != null)` のように実体があるか確認してから命令を送る手法も紹介されました。

### ガーベージコレクション (GC)
*   **使い捨ての顔の例え**：アンパンマンの顔が新しくなる（`new` される）たびに、古い顔（古いインスタンス）は不要になります。
*   Java/Processingでは、どこからも参照されなくなったメモリ上の不要なオブジェクトを自動的に回収・解放する「ガーベージコレクション」という機能が備わっています。

## 5. 応用：PImageクラスと画像利用
Processingに用意されている `PImage` クラスを利用して、外部画像（バイキンマンなど）を扱う方法が解説されました。

*   画像をドラッグ＆ドロップして `data` フォルダに保存。
*   `loadImage()` で画像を読み込み、`image()` 関数で表示する。
*   これも内部的には `PImage` というクラスのインスタンスとして扱われており、フィルター処理などのメソッドを利用可能です。
