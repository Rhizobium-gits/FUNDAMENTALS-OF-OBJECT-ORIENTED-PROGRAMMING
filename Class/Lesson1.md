# 授業内容まとめ：オブジェクト指向プログラミングとProcessingの活用

## 1. 授業の概要と進め方
*   **ターゲット**: Pythonなどでプログラミング経験があり、オブジェクト指向を「なんとなく」使っている人が、より深く理解することを目的としています。
*   **レベル設定**: 主に中高でプログラミング経験が少なかった学生を想定した内容です。
*   **授業構成**: 前半はZoomでのレクチャー、後半（約1時間）は各自で手を動かす課題の時間となります。
*   **受講形態**: 教室での対面受講とオンライン受講のハイブリッド形式です。教室では質問がしやすく、Zoomでは録画の共有が容易というメリットがあります。録画はKLMSにアップロードされます。

## 2. 開発環境：Processing
### 特徴と導入
*   **マルチ言語対応**: Java、JavaScript、Python、Shaderなど複数の言語で記述可能です。
*   **インストール**: [processing.org](https://processing.org/) からダウンロード。寄付によって運営されている財団法人のツールです。
*   **基本構造**: `setup()`（初期設定）と `draw()`（繰り返し描画）の2つの関数を基本とします。

### 推奨設定 (Preferences)
*   **言語**: 「Japanese」を選択し、日本語コメントを可能にします。
*   **エディタ機能**: 「コード補完」や「エラーの継続的チェック」をオンにすることで、タイポ（打ち間違い）を防ぎ、効率的に開発できます。
*   **サンプル**: 「Processing Handbook Second Edition」のサンプルコードをインストールして参照することが推奨されています。

## 3. オブジェクト指向プログラミング (OOP)
*   **概念**: 英語の「目的語（Object）」に由来し、オブジェクトに向かってプログラミングを行う考え方です。
*   **Javaの重要性**: 本授業ではJavaを使用します。JavaはAndroidアプリ（Kotlinの親戚）や、銀行などの大規模なサーバーシステムで今もメイン言語として使われています。
*   **汎用性**: この考え方は、Swift (iOS)、C# (Unity)、Python (AI) など、他の多くの現代的な言語にも共通して応用できます。

## 4. AIとプログラミングの未来
*   **AIの役割**: ChatGPTやClaudeなどのAIは「下書き」を生成するツールとして非常に強力です。
*   **人間に求められる能力**: 
    *   AIが生成したコードが正しいか判断し、修正する能力。
    *   全体を統合する「現場監督（アーキテクト）」としての知識。
    *   適切な指示（プロンプト）を出すためのプログラミングの基礎知識。
*   **技術の二極化**: 手書きでHTMLを書かなくなったのと同様に、単純なコーディング作業は減り、より高度な設計能力が重要になります。

## 5. 本日の課題：アンパンマンの描画
### ステップ1：基本描画
*   円（`ellipse`）や四角（`rect`）などの図形を組み合わせてアンパンマンを描きます。
*   `mouseX`, `mouseY` という予約語（変数）を使うことで、マウスの位置座標を取得できます。

### ステップ2：抽象化と関数化
*   描画処理を**関数（例：`drawAnpanman(x, y)`）**にまとめます。
*   **抽象化のメリット**:
    *   内部の細かい数値計算を隠し、「アンパンマンを描く」という一つの命令として扱えます。
    *   一度関数化すれば、引数（座標など）を変えるだけで、複数のアンパンマンを簡単に描画したり、マウスに追従させたりできます。
*   **命名の重要性**: 「目（eye）」や「目たち（eyes）」など、適切な名前を付けることで、より高次のプログラミング思考が可能になります。

## 6. 評価と提出物
*   **評価基準**: 毎週の課題提出（コード＋日本語コメント）の積み重ねで評価されます。
*   **配点**: 週を追うごとに難易度と配点が上がります。最終課題の内容が「S」評価に直結します。
*   **コメントの義務化**: AIに書かせた場合でも、必ず自分で理解し、コードの内容を日本語コメントで説明することが求められます。
